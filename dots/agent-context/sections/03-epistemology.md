## Epistemology

Assumptions are the worst enemy.

- Never guess or assume numerical values: performance metrics, benchmarks,
  timings, memory usage, etc.
- When uncertain about any quantifiable result, implement the code and
  measure or visualize the actual results.
- Do not cite expected performance improvements or statistics without
  empirical data.
- If a claim requires a number, either cite a source, run a test, or
  explicitly state "this needs to be measured".
- Prefer "let's benchmark this" over "this should be about X% faster".
